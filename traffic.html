Sanketa Technology Deep Dive

An edge-based, solar-powered AI traffic signal system designed for Bharat’s high-growth urban environments.

1. The Challenge

India’s rapidly growing urban corridors face unpredictable traffic, diverse vehicle classes, and outdated fixed-timer signals. These outdated systems increase waiting time, fuel wastage, emissions, and unsafe driving behavior due to a lack of real-time countdown transparency.

2. System Objectives

  • Real-time adaptive signal timing using multi-sensor fusion.
  • Reliable offline operation powered by solar-backed edge computing.
  • Priority routing for emergency services and public transport.
  • Public-facing countdown feeds for navigation apps and commuters.
  • Reduction in idle emissions, noise, and corridor delays.

3. Architecture Overview

  • Edge Controller: AI-enabled hardware fusing camera, radar, and loop sensors. Models run locally.
  • Sensing: Vision for classification, radar for all-weather detection, loops for redundancy.
  • Connectivity: Secure MQTT/gRPC uplink with fallback redundancy.
  • Central System (Optional): Aggregated analytics, remote OTA updates, fleet-wide optimization.
  • APIs: Internal optimization APIs and public SPaT (Signal Phase & Timing) feeds.
  • Storage: Time-series logs, video clips, and ML feature stores.
  • Dashboard: Live maps, queue heatmaps, manual overrides, alerts, and audit logs.

4. How the System Works

  1. Solar Power Array: Powers the entire smart pole, ensuring continuous operation.
  2. High-Resolution Vision System: Captures live traffic video for AI processing.
  3. AI Traffic Management Unit: Predicts queues, optimizes timings, and identifies emergencies.
  4. Smart LED Signal Modules: Dynamically adjust green/red timings based on real-time data.
  5. Data Uplink Module: Sends performance, health, and traffic metrics to the dashboard.
  6. Wireless Antenna: Enables V2I communication, coordinated corridors, and priority routing.

Every subsystem works in harmony — sensing, decision-making, communication — to deliver safer, smoother, and more efficient urban mobility.

5. Data Inputs

  • Real-time: Occupancy, speeds, queue lengths, pedestrian requests, weather, incidents.
  • Historical: Patterns, seasons, events, baseline congestion models.
  • Derived: Arrival variability, spillback prediction, pedestrian density.

6. AI & Optimization Strategy

The system uses a hybrid AI model combining deterministic traffic safety rules with machine-learning prediction.

  • Arrival Prediction: Traffic arrival forecasting and clearance estimation.
  • Phase Optimization: Multi-agent reinforcement learning, validated via shadow mode.
  • Safety Layer: Ensures compliance with min/max green, pedestrian clearance, and emergency response.
  • Simulation Loop: SUMO-based offline playbacks detect drift and trigger retraining.

7. Countdown Transparency

SPaT (Signal Phase & Timing) updates are published every 100–500 ms, normalized with confidence scoring. Public APIs provide real-time phase, remaining seconds, and fallback to static estimates when prediction confidence is low.

8. Deployment Phases

  1. Pilot at 5–10 intersections with baseline data capture.
  2. Shadow adaptive mode testing during off-peak hours.
  3. Corridor-level rollout with green-wave synchronization.
  4. Citywide scaling and public developer API launch.
  5. Continuous optimization with incident prediction and eco-routing.

9. Key Performance Metrics

  • Average delay & stop count per vehicle.
  • Queue percentiles & spillback events.
  • Travel time reliability and buffer index.
  • Emission reductions (CO₂, NOx).
  • Pedestrian wait times & compliance levels.
  • API uptime, signal latency, and response time.
  • Safety indicators (hard braking, red-light violations).

10. Risks & Mitigations

  • Sensor Failure: Multi-sensor redundancy and health monitoring.
  • Model Drift: Scheduled validation and fallback logic.
  • Privacy: Edge anonymization and minimal data retention.
  • Cybersecurity: Segmented network, mTLS, continuous testing.
  • Regulatory: Early coordination, transparent public metrics.

11. Next Actions

  • Stakeholder workshops and intersection mapping.
  • Sensor deployment and baseline data collection.
  • Adaptive logic A/B testing in shadow mode.
  • SPaT API public release for developers.
  • Policy and governance documentation.

12. Impact Summary

Adaptive signal systems typically reduce delays by 10–30% and emissions by 5–15%, delivering compounding benefits across connected corridors and smart mobility ecosystems.
